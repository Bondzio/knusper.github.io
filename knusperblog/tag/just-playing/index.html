Firefox - GTK Dark Theme Fix

Problem:

Using a dark KDE (or GTK native) color theme (e.g. zenburn) causes content in form input elements to be illegible. This known Firefox bug basically renders the browser unusable on many webpages, that do not come with custom CSS rules for their forms.

Solution:

Use the Stylish Firefox add-on to create a custom global rule no dark forms and buttons:

    input:not(.urlbar-input):not(.textbox-input) {
        -moz-appearance: none !important;
        background-color: white;
        color: black;
    }

    #downloads-indicator-counter {
        color: white;
    }

    textarea {
        -moz-appearance: none !important;
        background-color: white;
        color: black;
    }

    select {
        -moz-appearance: none !important;
        background-color: white;
        color: black;
    }

Succsesfully tested with Firefox version 31.

Source: Arch Linux Wiki – Retrived 19.08.2014

Matplotlib Animation as Animated GIF

First save matplotlib animation in python code with:

ani.save('animation.mp4', fps=10)

Use ffmpeg to decode single frames to gif:

mkdir gifs && ffmpeg -i animation.mp4 -r 10 gifs/%04d.gif

Use gifsicle to create animated gif:

gifsicle --use-colormap 'bw' --delay=10 --loop ./*gif > animation.gif

See also this entry at dtbaker.net/random-linux-posts.
