bash autocd

To save typing, set in your .bashrc

shopt -s autocd

Now, you just do directory instead of cd directory.

Works with bash versions > 4. For more cool stuff, check Section 4.3.2 of bash manual.

Cubehelix as Default in ds9

I love Dave Green’s cubehelix color scheme for visualization of astronomical intensity datasets (e.g. images).

In fact, I love it so much, that I want it to be the default in SAOImage ds9. A little alias does the trick for me:

alias ds9='~/local/bin/ds9 -cmap file ~/local/lib/cubehelix00.ds9'

Here the file cubehelix00.ds9 must be obtained from this page.

IOP Science RSS Feeds Broken (affects ApJ, ApJS & AJ)

Since some time now the RSS Feeds of the high impact astronomy journals – The Astrophysical Journal, The Astrophysical Journal Supplement & The Astronomical Journal- do not work for me in KDEs feed reader Akregator. I believe the reason for this is, that the RSS feeds from the publisher (IOPScience) are not well formed. In particular I suspect the following malformed structure at the beginning of the feeds

<title>The Astrophysical Journal - latest papers</title>
<link>http://iopscience.iop.org/0004-637X/?rss=1</link>
<description>Latest articles for The Astrophysical
Journal</description>
<items>
  <rdf:Seq>
    <rdf:li />
    ...
    <rdf:li />
  </rdf:Seq>
</items>

According to the RSS 1.0 Specification the <rdf:li /> elements needs to be formed like <rdf:li resource="{item_uri}" /> were {item_uri} refers to the same value specified in each item element’s rdf:about={item_uri}.

So this list is something like a “Table of Contents” of the RSS file, and apparently Akregator evaluates this list. I informed IOPScience about the issue… Lets see what happens.

At the moment a possible workaround would be, to use a feed converter (e.g. Googles Feedburner) to convert the feeds to RSS 2.0 and then feed those converted feeds to Akregator. But I really hope, that IOPScience fixes the problem.
